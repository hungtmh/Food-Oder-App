TỔNG QUAN DỰ ÁN

  • Mục tiêu: Xây dựng ứng dụng Android đặt món online hoàn chỉnh.
  • Đối tượng:
    • Khách hàng: Duyệt menu, đặt món, thanh toán, đánh giá.
    • Admin: Quản lý món ăn, đơn hàng, doanh thu, AI.
  • Nền tảng: Android (Java).
  • Backend: Supabase (PostgreSQL + REST API).
  • Điểm nổi bật: Tích hợp AI phân tích cảm xúc & dự đoán xu hướng.

KIẾN TRÚC HỆ THỐNG

Mô hình: MVC (Model – View – Controller)

Kiến trúc: Thick Client – Thin Server (BaaS)

Xác thực:

  • Custom Auth
  • SHA-256 Hashing
  • SharedPreferences

Network:

  • OkHttp 4.12
  • Interceptor tự gắn API Key

THANH TOÁN SEPAY

Cơ chế hoạt động (QR Banking):

  1. Khách chọn "Chuyển khoản ngân hàng"
  2. Hệ thống sinh mã QR chứa thông tin chuyển khoản (Sepay API)
  3. Khách quét QR bằng app ngân hàng ➝ Chuyển tiền
  4. Khách bấm "Đã chuyển khoản"
  5. Server xác thực qua Webhook Sepay (Supabase Edge Function)
  6. Đơn hàng chuyển trạng thái thành "Đã thanh toán"
App ➝ Sepay API ➝ QR Code ➝ Quét & Chuyển ➝ Webhook ➝ Xác nhận đơn

GỬI THÔNG BÁO HÀNG LOẠT

  1. Nhập Tiêu đề chiến dịch.
  2. Nhập Nội dung (tối đa 500 ký tự có countdown).
  3. Xác nhận gửi thông báo hàng loạt.
  4. Hệ thống gọi Supabase Edge Function tạo broadcast.
  5. Báo cáo kết quả "Gửi thành công đến N khách".

Lưu ý: Hỗ trợ chống spam & Xóa form nhanh.

AI: SENTIMENT & TRENDS

Phân tích Cảm Xúc (NLP) Dự đoán Xu hướng
  • Lõi: HF BERT qua Edge Functions.
  • Phân loại: Tích cực, Trung tính, Tiêu cực.
  • Hỗ trợ xử lý song ngữ (Việt, Anh).
  • Tốc độ: ~5ms cho mỗi thao tác quét.
  • Phương pháp: Score-based Decision Tree.
  • Dựa trên lượng bán 30 ngày + Sentiment.
  • Nhãn 4 mùa:
    - Hot Seller (Bán chạy, được khen)
    - At Risk (Ế khách, bị phàn nàn)
    - Declining (Rớt hạng) | Stable (An toàn)
